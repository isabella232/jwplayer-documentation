JW Player's Tracks configuration options allow you to customize the experience you want your viewers to have with your content, including making it more accessible. Click here to see our demo of an interactive video transcript, one way in which you can take creative advantage of all Tracks have to offer.

Any of the track types discussed on this page can be implemented via WebVTT (VTT) files. In addition, Captions tracks can be added via an SRT file, though SRT will not be covered here.

VTT files are easy to understand, and can be authored with a simple text editor—as you will see in the examples below.

 

Using Tracks

  • Dashboard
  • Code

In the JW Player Dashboard, each content object has a "tracks" tab, where you can manage the tracks for that video.

  1. Caption tracks are special here, and can be uploaded, edited, or even created from a blank file using our captions editor.
  2. Already-existing Captions, Chapters, and Metadata tracks can all be uploaded here.
  3. Thumbnail tracks are automatically created when a video is uploaded into the JW Platform.

When setting up a player in code, you can take advantage of the "tracks" configuration block. There are four possible properties, that may or may not be needed depending on which type of track you plan to embed:

file (required)
URL to the WebVTT (or SRT, for captions) file to display (e.g. /assets/myCaptions.vtt). Note: these files are subject to Crossdomain Security Restrictions, in both Flash and HTML5 mode.
label
This option applies to captions, and is a human-readable track name (e.g. English Captions). If more than one track is available, this label will be displayed in the captions selection (CC) menu in the player.
"default"
This option applies to captions, which are Off by default and can be enabled using the player's CC menu. Set this to true if you want captions to display on startup. (Note: the option should be placed in quotes because default is a reserved JavaScript keyword in IE8.)
kind (required for Chapters, Thumbnails, and Metadata)
The type of text track being specified (captions, chapters, thumbnails, metadata). Captions are the default; however, it is good practice to add this property anyway for readability when captions are being used.

var playerInstance = jwplayer("myElement")
playerInstance.setup({
    playlist: [{
        file: "/assets/sintel.mp4",
        image: "/assets/sintel.jpg",
        tracks: [{ 
            file: "/assets/captions-en.vtt", 
            label: "English",
            kind: "captions",
            "default": true 
        },{ 
            file: "/assets/captions-fr.vtt", 
            kind: "captions",
            label: "French"
        }]
    }]
});

Each "kind" of tracks file has different format expectations within the file, as outlined below.

Caption File Formats

When a Captions file is associated with a video (via the dashboard or the embed block), viewers see text overlaid on the video.

JW Player supports both VTT and SRT files for captions across all browsers, in both Flash and HTML5 mode. However, VTT captions are strongly preferred, since they are part of the HTML5 standard. Note: Captions will not work on iOS for versions prior to JW Player 7.3. For 100% caption support on iOS, update to JW Player 7.3.

JW Player also provides more captions information here:

  • HLS Captions (CEA-608)
  • Styling for FCC Captions Requirements

Captions VTT Example

WEBVTT 00:00:12,000 --> 00:00:15.000 What brings you to the land of the gatekeepers? 00:00:18,500 --> 00:00:20.500 I'm searching for someone. 00:00:36,500 --> 00:00:39.000 A dangerous quest for a lone hunter. 00:00:41,500 --> 00:00:44.000 I've been alone for as long as I can remember.

Embedding a Captions File

Here is an example setup, using one video and two VTT files. The English captions will render by default:

var playerInstance = jwplayer("myElement")
playerInstance.setup({
    playlist: [{
        file: "/assets/sintel.mp4",
        image: "/assets/sintel.jpg",
        tracks: [{ 
            file: "/assets/captions-en.vtt", 
            label: "English",
            kind: "captions",
            "default": true 
        },{ 
            file: "/assets/captions-fr.vtt", 
            kind: "captions",
            label: "French"
        }]
    }]
});

When a Chapters track is associated with a video (via dashboard or the embed block), a viewer can quickly hover over the cue point and can see the title of the chapter or section. Clicking on the cue point navigates the viewer directly to the selected point in the video.

Chapters VTT Format

The following required and optional elements make up a valid Chapters VTT file:

Cue identifier (optional )
Used to identify the actual cue point.
Cue timings (required)
Specifies the time zone for the cue. The chapter will be displayed on the start of the zone.
Chapter title (required)
The actual title you want displayed.

Chapters VTT Example

WEBVTT

Chapter 1
00:00:00.000 --> 00:01:42.000
Opening credits

Chapter 2
00:01:42.000 --> 00:04:44.000
A dangerous quest

Chapter 3
00:04:44.000 --> 00:05:50.000
The attack

Notes:

  • The title that gets displayed in the tooltip is the text located beneath the time.
  • VTT files should always be saved in UTF8 encoding to prevent character issues.
  • VTT files are subject to cross-domain security restrictions and therefore won't automatically load from another domain than the player. See Crossdomain File Loading for more info.

Embedding a Chapters File

After you have your VTT file with chapter cues, you can upload it with the video it belongs to in the JW Player dashboard.

If you are not hosting your content with JW Payer, or you'd like to add this track by hand, add the following tracks configuration block to your player embed, with the kind set to chapters.

The full player embed code for above live example looks like:

var playerInstance = jwplayer("container");
playerInstance.setup({
  file: "/upload/sintel.mp4",
  image: "/upload/sintel.jpg",
  tracks: [{
    file:'/upload/chapters.vtt',
    kind:'chapters'
  }],
  width: 640,
  height: 270
});

Note the timeslider skin element can be customized. See the Skin CSS Reference for more info.

Chapters can also be loaded as part of RSS feeds. See the RSS Feeds Reference for more info.

Thumbnails VTT Format

When used for thumbnails, VTT files contain links to the actual thumbnail images, which can be in JPG, PNG or GIF format.

The structure of the Thumbnails VTT file is straightforward. It consist of a list of cues, which each include:

  • The range the tooltip thumbnail represents. Note the range needs to be in (HH:)MM:SS.MMM format. Only this exact notation will be parsed.
  • The URL to the tooltip thumbnail for this range. The URL is relative to the VTT file (not to the page or player), much like images included in CSS sheets.

Note: JW Player renders tooltip thumbnails in their original dimensions. We recommend 100x150 pixels — Smaller thumbnails are hard to decipher and larger thumbnails tend to distract from the main content.

Thumbnails VTT Example

Here is an example VTT file. It belongs to a 20-second video, for which 4 preview thumbnails are included:

WEBVTT

00:00.000 --> 00:05.000
/assets/preview1.jpg

00:05.000 --> 00:10.000
/assets/preview2.jpg

00:10.000 --> 00:15.000
/assets/preview3.jpg

00:15.000 --> 00:20.000
/assets/preview4.jpg

Using Sprites

For many reasons (file sizes, load delays, server requests, etc.), having each thumbnail be a separate image is sub-optimal. Therefore, JW Player supports Thumbnail Sprites: multiple thumbnails stitched together into a single image. Here's an example:

In the VTT file, the individual thumbnails can be located by adding their coordinates to the thumbnail URL using a spatial Media Fragment:

WEBVTT

00:00.000 --> 00:05.000
/assets/thumbnails.jpg#xywh=0,0,160,90

00:05.000 --> 00:10.000
/assets/preview2.jpg#xywh=160,0,320,90

00:10.000 --> 00:15.000
/assets/preview3.jpg#xywh=0,90,160,180

00:15.000 --> 00:20.000
/assets/preview4.jpg#xywh=160,90,320,180

Note: JW Player supports pixel-based fragments, not percentage-based ones.

Embedding a Thumbnails File

Here is an example setup, loading a single video with Tooltip Thumbnails:

var playerInstance = jwplayer("myElement");
playerInstance.setup({
    playlist: [{
        file: "/assets/myVideo.mp4",
        image: "/assets/myPoster.jpg",
        tracks: [{ 
            file: "/assets/myThumbnails.vtt", 
            kind: "thumbnails"
        }]
    }]
});

RSS Feeds

When using external RSS playlists, Tooltip Thumbnails can be included using a <jwplayer:track> element. Here is an example feed, containing the same video and thumbnail track as above:

<rss version="2.0" xmlns:media="http://rss.jwpcdn.com/" > 
  <channel> 

    <item> 
      <title>Sintel</title>
      <description>Sintel is a fantasy CGI from the Blender Open Movie Project.</description>
      <jwplayer:image>/assets/myPoster.jpg</jwplayer:image>
      <jwplayer:source file="/assets/myVideo.mp4" />
      <jwplayer:track file="/assets/myThumbnails.vtt" kind="thumbnails" />
    </item>

  </channel>
</rss>
