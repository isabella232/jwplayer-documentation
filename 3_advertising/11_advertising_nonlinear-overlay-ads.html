In addition to standard, linear video advertising, JW Player also supports a method of displaying ad banners as an overlay on video content. This means that advertisements can appear in-line without disrupting playback. This method of advertising is known as a non-linear overlay.

Non-linear ads are configured via VAST in a similar way to linear ads. Within the creative portion, instead of specifying a linear ad type, a nonlinear type is used. A brief example of this configuration can be seen below:

<Creative sequence="1">
<NonLinearAds>
<NonLinear height="50" width="300" maintainAspectRatio="true" scalable="true" minSuggestedDuration="00:00:30">
<StaticResource creativeType="image/jpeg">
overlay.jpg
</StaticResource>
<NonLinearClickThrough>http://www.jwplayer.com/</NonLinearClickThrough>
</NonLinear>
</NonLinearAds>
</Creative>

Similar to linear ads in our static VAST article, JW Player looks for certain elements within a VAST tag creative to build an ad experience. In the above code, we can see the size of the advertisement, the picture file being loaded, where our clickthrough will direct to, and the length of time that this resource will appear for. Additionally, with the minSuggestedDuration portion, we also dictate how long this linear ad should appear for. (In this case, the overlay will show up for 30 seconds during content.)

The creative that we've specified above is implemented in the player below. We've also configured an offset of 5 seconds before displaying the ad. Using this offset configuration, nonlinear ad tags can be displayed at any time throughout a content file. Finally, we specify the ad type as a nonlinear ad. This allows the ad to play over content in an uninterrupted way. Also, ad markers will only appear for linear ads.

 
var playerInstance = jwplayer("myElement");
playerInstance.setup({
image: "assets/image.png",
file: "assets/video.mp4",
	advertising: {
	offset: 5,
	client: "vast",
		schedule: {
			break1: {
			tag: "assets/overlayAd.xml",
			type: "nonlinear"
			}
		}
	}
});
