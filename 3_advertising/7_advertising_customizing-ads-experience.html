You can set an amount of time that you want your viewers to watch an ad before being allowed to skip it. When this skip offset is configured, a countdown will appear in the player and become active when the time has passed. This skip cancels the ad playback and does not fire any ad complete trackers.

Please note that this option is configurable for VAST only and not Google IMA.

If you are using a VAST 3.0 tag, any skipoffset specified in your ad tag will supercede any player-specific configuration options for skipping ads.

​
The following configuration options are available:

skipoffset
Set this to an amount of time in seconds that you want an ad to be watched before it can be skipped.
skipmessage
This is used to provide a customized countdown message. Use the form Skip ad in xx seconds..
skiptext
This sets the text of the Skip button after the countdown is over..

Example Preroll with Skip

To set a skip ad offset all you need to do is add the following to your advertising configuration block.

skipoffset
Set this to an amount of time in seconds that you want an ad to be watched before it can be skipped.
 

Sample embed code

The skipoffset can only be set at the advertising block level and not for individual ads in an ad schedule.

var playerInstance = jwplayer("myElement");
  playerInstance.setup({
    image: "/uploads/myPoster.jpg",
    file: "/uploads/myVideo.mp4",
    primary: 'flash',
    advertising: {
      client: "vast",
     'skipoffset':5,   
      tag: "//adserver.com/vastResponse.xml"
    }
  });

During playback of linear ads with VAST, JW Player displays a friendly ad message to notify viewers. However, via a simple customization option, it is possible to set a custom ad message. This article explains the process.

By default, JW Player's Ad message is 'This ad will end in xx seconds'. However, because of langauge differences, desired branding, or if VAST tags are being configured for other means, this message is fully customizable. Our default message appears as follows:


However, this message can be modified by setting the admessage configuration option inside of the advertising block. The string xx in this option will also automatically be replaced with a countdown. The use of the xx is entirely optional however. An example of a custom ad message configuration can be seen below:

 
var playerInstance = jwplayer('myElement');
playerInstance.setup({
file: 'assets/myVideo.mp4',
  advertising: {
  client: 'vast',
  tag: '//adserver.com/vastResponse.xml',
  admessage: 'This is totally not an ad that will end in xx seconds.'
}
});

JW Player supports the insertion of companion ads into the page. HTML/IFRAME and GIF/JPG/PNG ads are supported for both VAST and IMA.

In order to render a companion, you must set the id, width and height of the DOM element the companion should be placed into. JW Player will then insert a companion if the ad tag contains one that exactly matches the width and height provided in the configuration. If the dimensions do not match up, JW Player will not display an ad. (which could lead to broken page designs)

We can confirm our companion dimensions by viewing our VAST file itself. In the companion section, we can confirm:

<Companion width="300" height="250">

This means that our tag will attempt to load a companion with a width of 300 and a height of 250. In addition, the div where we'd like to set this up has been given the ID of rectangle. Within our advertising block, we will go ahead and set our companiondiv portion with the aforementioned information:

advertising: {
client: "vast",
tag: "myVast.xml"
    companiondiv: {
        id: "rectangle",
        width: 300,
        height: 250
        }
}
