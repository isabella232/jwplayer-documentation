Use the new JW Player interface to create cloud-hosted ad schedules.

With JW Player you can schedule an ad to appear anytime during playback. You can use JW Player Ad Scheduling to create a TV-style viewing experience by inserting ads at opportune moments in your long-form content. An intuitive dashboard interface allows you to create and manage ad schedules that can be easily paired with your content and your cloud-hosted players. You can also easily use a schedule to add non-linear ads to overlay on top of the video during playback.

Creating an Ad Schedule

To create an ad schedule all you need is a set of tags from your ad partner. You can use the same tag for each ad break.

  1. Navigate to the Ads page.
  2. Create a New Ad Schedule or edit an existing one.
  3. Choose an Ad Manager - VAST or Google IMA.
    1. If you are using DFP it is better to select Google IMA.
  4. Enter a friendly name for your schedule.
  5. Press at your desired ad break
    1. Preroll
    2. Midroll
    3. Postroll
  6. In the drop down menu underneath Tag Name choose Add New Tag or select an existing tag.
    1. If you add a new tag, it will automatically be saved to your Ad Tag manager and you can choose it for later break or a different schedule
  7. Enter in a friendly name for your tag.
    1. Example: AdX Preroll
  8. Enter in the url to the ad tag.
    1. Use Ad Tag Targeting Macros to get better fill on your ads.
  9. Press Save within the Ad Break
  10. Add as many breaks as needed.
  11. Make sure to give your schedule a friendly name and Save.

Notes and Tips

  • Apply skip offsets with each ad break to increase engagement.
  • You can create as many ad schedules that you need.
  • Use percentage time offsets for midrolls for a generic schedule.
  • Only midroll ads can be forced to fill with a non-linear overlay.

Associating a Schedule with a Player

Now that you have created an ad schedule, you need to associate with a Player. Note that if you do not have any schedules created, the option for JW Player Schedule will not appear. Navigate to the Player section in your Dashboard.

  1. Click New Player or edit an existing Player.
  2. Set your desired playback settings and other customizations.
  3. Open the Advertising pane on the left-hand-side.
  4. Turn Advertising on.
  5. Choose JW Player Schedule
  6. Select your schedule from the drop down menu.
  7. Set the text for the user interface elements that you desire.
  8. Save your Player
  9. Publish your player with content to begin using the ad schedule.

Managing your Ad Tags

All of your ad tags are automatically saved so you can use them later. To manage the tags navigate to the ads section and click on Manage Tags.

On this page you can do the following actions:

  1. add new ad tags,
  2. edit existing tag names and tag urls,
  3. and delete tags.

To add a new ad tag, press Create New Tag. Enter the name and tag url and press Save. To edit an existing tag, hover over the name or tag url and click to edit. Make sure to save changes.

To delete tags, select the tags you want to delete by clicking on the checkbox. Once you have selected at least one tag, you can delete it by pressing the Delete button.

JW Player Setup Ad Schedule (VAST, Google IMA, FreeWheel Plugins)

The ad schedule allows for scheduling ads against the main video content directly in your player embed code. A schedule is required whenever you want to run anything beyond a basic pre-roll (for which you can use a tag). This section explains in detail how the ad schedule works.

The schedule block contains one or more ad breaks. Each ad break specifies a point on the timeline of the main content at which an ad must be inserted. A typical schedule looks like this:

schedule: {
    adbreak: {
        offset: 'pre',
        tag: '//adserver.com/vastResponse1.xml'
    },
    60seconds: {
        offset: 60,
        tag: '//adserver.com/vastResponse2.xml'
    },
    postroll: {
        offset: 'post',
        tag: '//adserver.com/vastResponse3.xml'
    }
}

Each ad break in a schedule has a unique name, an offset, a tag and an (optional) type:

offset
This required option describes the point in time at which to play the ad. Values permitted include the words pre and post, or an offset in seconds:
  • pre: specifies that the ad should be played before the video content.
  • post: specifies that the ad should be played after the video content.
  • A time value specified in one of the following formats:
    • seconds: '50'
    • fractional seconds: '50.5'
    • percentage of the entire video: '50%' (NOTE: Google IMA does not support percentage offsets.)
    • timecode(hh:mm:ss.mmm): '01:55:30:000'
tag
This required option is the URL to the ad tag, which contains the VAST, IMA, or FreeWheel response. VAST tags can contain one or more ad tag variables. See (the bottom of) Supported Ad Formats for details. Note: FreeWheel does not require URLs, but custom strings as configured in the FreeWheel MRM platform.
type
This option is only required when loading a nonlinear ad tag (a.k.a. overlay). The option must then be set to nonlinear. The default type is linear.

Note: Your ad schedule is applied to each playlist item. If you have a multi-video playlist, ads will be repeated for each entry with the same schedule. To have a different schedule of ads per video in a playlist, see the section below on scheduling ads in a playlist.

Example

The following shows a player and its configuration. This includes a schedule for one preroll and a nonlinear midroll at 5 seconds:

 
var playerInstance = jwplayer('myElement');
playerInstance.setup({
file: 'assets/myVideo.mp4',
	advertising: {
		client: 'vast',
		schedule: {
			adbreak1: {
				offset: "pre",
				tag: '//adserver.com/vastResponse1.xml'
			},
			adbreak2: {
				offset: 5,
				tag: '//adserver.com/vastResponse2.xml',
				type: 'nonlinear'
			}
		}
	}
});

Note that you must not set the overall advertising tag option when setting a schedule. Each ad break uses and requires a different tag. The tag option without a schedule block will default to a pre-roll ad.

Ad Schedule Per Playlist Item (VAST and FreeWheel Clients-only)

Sometimes it is necessary to have an ad schedule that is unique per video in a playlist. This setup will allow to have a default schedule be applied to videos in a playlist that do not have a schedule defined and a specific set of ads used for an individual video when needed. To set this up, use the adschedule configuration within the playlist setup block. Please note that this feature is only available when our VAST or FreeWheel clients are being utilized.

adschedule
This is an optional schedule block used within a playlist configuration. It allows the JW Player to use a specific set of ads for the respective video in the playlist. If this is not set for a particular playlist item, the player will fall back to the advertising schedule block.

Example

The following example is a playlist of two videos. The first video has an adschedule block configured and the second video does not. Because of this, the second video will default to the schedule defined in the advertising block:

 
var playerInstance = jwplayer('myElement');
playerInstance.setup({
	'playlist': [{
		'file': "video1.mp4"
		'title': 'Ad Scheduled Video',
			'adschedule': {
					'adbreak1': {
					'offset': "pre",
					'tag': "preroll-1.xml"
				},
				'adbreak2': {
					'offset': 5,
					'tag': "midroll-1.xml"
				}
			}
		},{
		"file": "video4.mp4"
		}],
	'advertising': {
		'admessage': "Your video won't play for xx seconds",
		'client': "vast",
			'schedule': {
				'adbreak1': {
				'offset': 'pre',
				'tag': 'preroll-2.xml',
				'skipoffset': '5'
			}
		}
	}
});
