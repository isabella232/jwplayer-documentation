JW Platform allows you to easily trim any video in your video library (VOD or live archive). This is especially useful for live streams created using JW Live. Once your live event over, you can now trim any preceding or post content that you want to discard.

To trim a video, simply select any video from your library. This will take you to the video detail page (see below). 
NOTE: Only videos that have MP4 format originals with the H.264 codec and ".mp4" file extension can be trimmed.

Click the “Trim” button next to the Duration value (see below).
 


This will open up the trimming popup. Here it’s really easy to set the new “start time” and “end time” of your video by either dragging each end of the blue control bar (see below) or by entering manual timecode values into the “Start Time” and “End Time” text boxes.
 
 


Once you are happy with your trim, click the “Trim and Save” button to save your changes. This will kick off transcoding jobs that will reprocess all output conversions. Please allow adequate time for your trimming transcodes to complete. Time varies depending on the length of your content.

NOTE: Your original video will NOT be truncated, only the conversions will inherit the new “trimmed” duration (for MP4, HLS, DASH, etc.). That way, if you want to re-trim your video later you can because the original will persist. This way, you can re-trim your video as many times as you want without losing any data from the original. Also note, to access your original you must use the API. The original is not available for download from within the portal.
 

API Access

You can also trim your videos through the JW Platform API. The /videos/create and /videos/update calls both support the “trim_in_point” and “trim_out_point” parameters.

Here are example API calls using the Python SDK client to upload/create a video: 
/videos/create : client.request('/videos/create', {'trim_in_point': '00:00:00.000', 'trim_out_point': '00:03:59.000'})
Here are example API calls using the Python SDK client to update an existing video: 
/videos/update: client.request('/videos/update', {'update_file': True, 'video_key': 'iK5PWlIl', 'trim_in_point': '00:00:00.000', 'trim_out_point': '00:02:59.000'})
