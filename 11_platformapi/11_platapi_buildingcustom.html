Most JW Platform customers use our convenient single-line codes for embedding video players. However, it's also possible to write custom embed codes, e.g. when embedding content from outside JW Platform or when rendering highly dynamic player embeds.

This document applies to users of JW Platform. Don't have Hosting? Upgrade Now.

Use Cases

In the majority of use cases, JW Platform's standard single-line embed codes are the best option for embedding players with a video or playlist. These embed codes can be found in the dashboard or generated using the API.

There are cases where more control over the JW Player configuration is required. For example when:

  • Embedding content not managed by JW Platform (e.g. a YouTube video).
  • Rendering highly dynamic player configurations (e.g. user captioning preferences).

These cases can be implemented with JW Platform by using player libraries instead of player embeds.

Where To Find

Player libraries contain just the core JW Player JavaScript code, without any of the embed-specific configuration options. Embedding them does nothing yet to your page, but all the plumbing is available to write your own custom JW Player embeds. They are essentially the same as a copy of the self-hosted JW Player.

The player library JavaScripts have the following URL structure:

http://content.jwplatform.com/libraries/PLAYERKEY.js

The PLAYERKEY is different for every player created in your dashboard. Both this key and the URL to a player's library can be found in the dashboard details page of every player:

Example Setups

Here is an example HTML page for the first case. It uses a player library to embed a YouTube Video. The embed still loads the video from JW Platform (using an RSS feed), but sets a custom sharing link for every pageview:

<html>
<head>
  <script src="//content.jwplatform.com/libraries/S2xWVQ9r.js"></script>
</head>

<body>
  <div id="container"></div>
  <script>
    jwplayer("container").setup({
      file: "http://www.youtube.com/dQw4w9WgXcQ"
    });
  </script>
</body>
</html>

Here is an example HTML page that uses a player library to build an embed with custom captions settings. You could imagine these settings to be user preferences and therefore be different for every viewer:

<html>
<head>
  <script src="//content.jwplatform.com/libraries/S2xWVQ9r.js"></script>
</head>

<body>
  <div id="container"></div>
  <script>
    jwplayer("container").setup({
      playlist: "//content.jwplatform.com/feeds/F2YKWOWd.rss",
      captions: {
        color: "FFCC00",
        backgroundColor: "000000",
        backgroundOpacity: 50
      }
    });
  </script>
</body>
</html>

Note this embed still loads the video from JW Platform, using an RSS feed to load both video sources, poster image, text tracks and metadata with a single line.

Advantages over Self-Hosting

To seasoned JW Player developers, it might look like player libraries are similar to downloading and hosting your own copy of the JW Player library. The workflow is the same, but there are several advantages to using player libraries from JW Platform:

  • A player library contains all configuration options from the dashboard as embedding defaults. This means it's still possible to centrally update custom embeds, as long as the option you are setting is not overruled in the custom embed. For example, you could update the skin or advertising tag of a player in the dashboard to have them auto-updated across all your embeds.
  • Another key feature that continues working with player libraries is engagement analytics. When embedding a JW Platform video like the example above, engagement analytics (as well as plays and time watched) are still collected and reported for that video. This is not the case with self-hosted libraries.
  • Last, player libraries are delivered from our high-performance, high-availability Contentserver. It leverages various global CDNs to deliver over 10 Billion assets per month with over 99.99% availability. HTTPS is supported, as well as DNS masking and URL tokening. There's nothing to code or maintain; all is ready to go!
