JW Player supports embedding on Facebook with a separate "Facebook-specific" player. Before implementing this feature, please note that this lightweight player does not support the following:

  • Custom skinning
  • Advertising
  • HLS Streaming
  • API
  • Analytics
  • HTML5 Playback

If you are using JW Platform, these OG Tags will be generated automatically based on your internal platform metadata. (Your configured title within platform will generate the appropriate "title" OG tag, for instance) If you would prefer a more manual setup however, a step-by-step process is listed below.

Step 1 — Setting up your Facebook Player

Acquiring your Facebook player can be done by contacting JW Player support and requesting this. Note that this player is only available for currently subscribed JW Premium users and higher.

The JW Player for Facebook operates in a slightly different method than the standard JW Player 6. Here, we are capable of loading a setup configuration within a single URL, rather than utilizing javascript exclusively. The only required configuration option here is the file option. If we were to load a video file within the context of our player SWF, you would need two URLs. One would be the player.swf, and the other would be the video file itself. If our SWF file were located at the following URL:

http://mysite.com/jwplayer/player.swf

And we wished to load a video file located here:

http://mysite.com/myvideo.mp4

We would first need to encode the MP4 URL.

http%3A//mysite.com/myvideo.mp4

After doing this and combining the two portions, our resultant URL would appear as:

http://mysite.com/jwplayer/player.swf?file=http%3A//mysite.com/myvideo.mp4

If we were to test and access this URL directly, you should see only my player and its content.
As a final step, we'll add our autostart=true config option to allow automatic playback in a Facebook news feed. This is not a required configuration option, but it is a recommended one.

http://mysite.com/jwplayer/player.swf?file=http%3A//mysite.com/myvideo.mp4&autostart=true

If you're able to access your content and your player by accessing your version of the above URL, good job! On to step 2.

Step 2 — Configuring your OG Tags

OG (Open Graph) Meta Tags are the method of choice for relaying information to Facebook when content is shared. It's worth noting that this meta info is to be specified in the HTML that your player exists on. As such, the player is not responsible for things like Title, Description, or Thumbnails. Again, this is all dictated based on your OG tags on the web page itself. To understand how OG tags work, we highly recommend reading Facebook's page here.

By sharing a page with an embedded video, you'll likely want to to share the video itself on Facebook, and allow in-line playback on someone's news feed or wall.
We suggest using the following OG tags if possible:

<meta property="og:url" content="http://mysite.com/sharingiscaring.html">

The actual URL that will be shared on Facebook when someone shares your web page.

<meta property="og:title" content="Sharing on Facebook with JW Player!">

The title that will appear on a Facebook story when it gets shared.

<meta property="og:image" content="http://mysite.com/myposterimage.jpg">

Your poster image for the player itself.

<meta property="og:type" content="video">

The type of content that you're looking to share when sharing a page.

<meta property="og:video" content="http://mysite.com/jwplayer/player.swf?config=http%3A//mysite.com/myvideo.mp4&autostart=true">

The actual video content that will be shared as per Step 1. However, you will also need to configure a secure_url tag in addition to a video tag.

<meta property="og:video:secure_url" content="https://mysite.com/jwplayer/player.swf?file=https%3A//mysite.com/myvideo.mp4&autostart=true">

This is a newer requirement by Facebook. Content and components must be loaded via HTTPS in order to appear in-line on news feeds. Therefore, your config file and player.swf should be served via HTTPS.

<meta property="og:video:type" content="application/x-shockwave-flash">

Here, we specify the method of content loading. Since we're using a Flash-based SWF to load and play video content, this should be set to: application/x-shockwave-flash

<meta property="og:description" content="Want to know how to embed on Facebook? Here's the answer!" >

The description of the shared content. This will appear below your configured title.

<meta property="og:video:width" content="480">
<meta property="og:video:height" content="360">

The size that your video content will appear.

The next step involves the configuration of your JW Player's sharing block. By default, when a share action is performed, the page where the player is embedded will be the page that is shared. For any additional configuration options, please see our configuration page here.
