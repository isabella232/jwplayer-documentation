The Ads edition of JW Player has built-in support for the two leading video ad frameworks: VAST/VPAID and Google IMA. Not every VAST feature is supported at present. See Supported Ad Formats for more information.

This document applies to users of JW Player Ads. Don't have Ads?  Upgrade Now.

Requirements

In order to start running video ads in JW Player, you need the following three things:

  • A valid license for the Advertising edition of JW Player. The Free, Premium, and Enterprise editions do not support advertising. See our Pricing page if you are interested in running ads with JW Player.
  • A working embed of the Ads player on your website. Please refer to Embedding JW Player to get learn how to embed a basic player.
  • A working ad tag from an ad server, in either VAST/VPAID or Google IMA format. Again, see Supported Ad Formats for details.

Setup

Once your player is configured with desired options and playback behavior, it's time to set up your advertising. Ads are specified in the advertising configuration block. Inside, two options are required for a basic preroll ad to appear:

client
Set this to vast if you are running VAST/VPAID ads, or to googima if you are running Google IMA ads. Note you cannot mix both ad formats in a single player embed. Choosing a client will maintain the same client until the player is set up again.
Example: 'client': 'googima' or 'client': 'vast'
tag
Set this to the URL of the ad tag that contains the pre-roll ad. Only linear ads can be run as pre-rolls; if you want to run e.g. an overlay you need to set a schedule (more info below).
skipoffset
Set this to a numerical amount of time you want an ad to be watched before it can be skipped. More information on ad skipping is located here.

In addition to these two options, the schedule, companiondiv and admessage options are available for more elaborate setups which are discussed later on.

VAST Example

First, here is a basic advertising setup using a VAST/VPAID pre-roll:

var playerInstance = jwplayer('myElement');
playerInstance.setup({
  file: 'assets/myVideo.mp4',
  primary: 'flash',
  advertising: {
    client: 'vast',
    tag: '//adserver.com/vastResponse.xml'
  }
});

Google IMA Preroll

Here is an example of a setup of JW Player with a TrueView pre-roll advertisement served by Google IMA:

var playerInstance = jwplayer('myElement');
playerInstance.setup({
  file: 'assets/myVideo.mp4',
  primary: 'flash',
  advertising: {
    client: 'googima',
    tag: '//ad.doubleclick.net/pfadx/ABCDEFGH'
  }
});

Note that VAST and IMA ads don't require Flash, but we strongly recommend to set it as the primary rendering mode. Many ad servers (CORS) and creatives (FLV) are not yet compatible with HTML5. In addition, JW Player only supports VPAID ads in Flash. This ensures that a user viewing with Flash installed will receive the advertisement.

A note on Google IMA and iPhones

As per Google IMA guidelines, if ad skipping is configured for a particular ad, Google will not display this advertisement on iPhone devices. Since fullscreen video is utilized for video ads, and there is no way to instantiate an ad skip, the advertisement will be omitted.
