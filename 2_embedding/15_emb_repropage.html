When troubleshooting an issue with our player, it is very helpful for us to have a reproduction page (also referred to as a test page) where we can reproduce the issue you are seeing. Once we can reliably reproduce your issue, we can get to work determining the cause. And then if we have any suggestions or code corrections, for example, we can then send you back a link to an updated page. We can also use this test page to escalate any potential bugs to our Engineering team.

We can usually see what you are trying to do by reading your code, but it is also helpful to know your exact reproduction steps. We would also like to know what it is you are expecting the player to do, as well as as description of what the player is actually doing.

 

What does a reproduction page include?

  • The JW Player Javascript library URL you are using to load our player, and if necessary, the self-hosted license key
  • The HTML <div> tag for our player
  • Your jwplayer().setup()
  • List of reproduction steps, expected behavior, and actual outcome
 

We try to be experts when it comes to our player, but we cannot help you debug any code on the page that does not belong to us. Therefore it is also important to note that there are a few things that should not be included in your reproduction page.

 

What does a reproduction page NOT include?

  • Any other Javascript code or libraries not required for our player, like jQuery or require.js
  • Extra CSS code or stylesheets 

Here is a template for you to use when making a reproduction page:

<!doctype html>
<html>
<head>
<script src="http://your_cloud-hosted_player_URL_or_self-hosted_jwplayer.js"></script>
<!-- if necessary: <script>jwplayer.key="your_self-hosted_license_key";</script> -->
</head>
<body>
    <b>Issue:</b> Please indicate issue here
    <br><br>
    <b>Reproduction steps:</b> Please state the steps necessary to reproduce this issue
    <br><br>
    <b>Expected behavior:</b> Please state your expected behavior here
    <br><br>
    <b>Actual outcome:</b> Please state the actual outcome here
    <br><br><br>
    <div id=player>Loading your player...</div>
    <script>
        jwplayer('player').setup({
            file: 'your_video_URL',
        });
        
        //include any of your Javascript API calls here 
    </script>
</body>
</html>

By using a basic embed of our player, hopefully we can help isolate the part of your code that is causing the problem. We hope you find this to be a helpful and worthwhile troubleshooting step.
