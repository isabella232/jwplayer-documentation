This article will go step-by-step over how to set up your new JW Player Ads subscription. From the use of your key, to the implementation of the ad tag itself.

This document applies to users of JW Player Advertising. Don't have Advertising? Upgrade Now.

Step 1: Getting your key and player

The first thing you'll need to do, is log into your new JW Player account. The login page can be found here.
Upon logging in, you will need to visit your account page. Here, you'll see two generated items. There is the cloud-hosted player code, and the self-hosted key.

Please note that these two items are different and use different keys. Your cloud-hosted token is what you'll use if you choose to host your JW Player with us. If you choose to self-host, the Self-hosted player license key will need to be used in conjunction with the downloaded player here:



This will download a zip of all required files.

Step 2: Getting your player embedded

For a standard embed, there are three portions to have set up:

1 - The Javascript Embed - This portion of the embed is what associates the JW Player with the page you're working with. Depending on if the self-hosted or cloud-hosted is utilized, this portion of your code is all that will need to be changed.
2 - The div - This portion controls where your player appears on your page.
3 - The player embed - This portion references the aforementioned div and will control how your player appears. Your player setup options live here and will control what videos play, what ads are used, the size of the player, and many other items. We have a full list of configurable options here.Let's take a look at an example of each of these items!

1 - The Javascript Embed

This portion exists in the head of your page. As mentioned, there are two ways to do this. For the cloud-hosted player, your script will contain your player key, so there will only need to be one single line of code.

Cloud-Hosted Embed

<script src="http://jwpsrv.com/library/eeefffGGGhhh111jjj123.js" ></script>

For a self-hosted player, you will be the one hosting the files needed for this. As such, you'll need to use your self-hosted key from part 1. Assuming you were hosting your files in a folder called "jwplayer", your self-hosted embed might look something like this:

Self-Hosted Embed

<script src="/jwplayer/jwplayer.js" ></script>
<script>jwplayer.key="ABCdeFG123456SeVenABCdeFG123456SeVen==";</script>

Once this portion is done, we'll move on to step 2!

2 - The div

This is a pretty easy portion. Here, we just determine where on the page we want our player to appear. You'll need to set up a <div> element and choose its name. In this example, we'll set up a div element and call it "myElement". Any text we put in between the div will show up in the event that there's an error loading our player. In this case, we'll put in the text "Player load error!".

<div id='myElement'>Player load error!</div>

3 - The Player Embed

This is where we actually configure the player's options. For this example, we'll set up a player with a single video, a poster preview, and a preroll ad tag.

Let's take a look at the code we'll be using here. We can then break it down to each individual portion to see what's happening here.

​jwplayer('myElement').setup({
  file: 'assets/myVideo.mp4',
  image: 'images/myImage.jpg',
  primary: 'flash',
  advertising: {
    client: 'vast',
    tag: 'http://adserver.com/vastResponse.xml'
  }
});

Let's start from the top!

​jwplayer('myElement').setup({

​This portion of our code is invoking the setup of our player. Notice the 'myElement' portion here. This is the section that defines which div element we're going to generate our player in.

file: 'assets/myVideo.mp4',

The file portion of our embed references the video we wish to play. Here, the player will be looking for an mp4 file called 'myVideo' in the assets folder. This will change depending on how/where you're loading your content from.

​image: 'images/myImage.jpg',

Here is where we load our poster image. This is the image file that will appear before our video loads. Again, we're referencing a jpeg file called myImage in the images folder.

primary: 'flash',

By default, our player will attempt to load in HTML5 initially. With ads, we suggest trying to load in Flash first. Certain ad types (VPAID) require the use of flash, so this is always a preferred method of loading things. Note that this will not disable HTML5 mode, it will simply attempt to use HTML5 as a backup, instead of as a primary. In the case of iOS or Android devices which do not have flash, nothing will change with the player, since they can only load HTML5.

​ advertising: {
    client: 'vast',
    tag: 'http://adserver.com/vastResponse.xml'
  }

Here is where we set up our advertising. Since there are multiple advertising options, we try to contain these individual pieces within a "block". Notice that all of these advertising options are contained within an additional set of braces and designated with the "advertising" label.

client: 'vast',

Here, we specify the type of ad client used. Unless you are using Google's DFP, vast should be used here. You can usually tell you're using a Google DFP ad if your tag begins with "pubads.g.doubleclick.net". In this case, 'vast' will need to be changed to 'googima'.

tag: 'http://adserver.com/vastResponse.xml'

Here is where we specify our ad tag. This should be given to you by your ads provider.

​});

And we have now closed out our JW Player embed.

Step 3: Testing it out!

With all of the elements in place, you should now be able to test your page. Either utilizing a localhost setup, or a live web server, load your page up and see if the player appears. If you're seeing the text we've used above between our div (Player load error!), there was possibly a problem with your script or embed. Make sure all files are being referenced correctly. Also, it's important to add or subtract commas as needed. For each embed option, a comma must be used if there is another following it. For example:

client: 'vast',

has a comma since it is followed by

tag: 'http://adserver.com/vastResponse.xml'

Notice that the tag portion does not use a comma afterwards since it is the last item in the embed.
