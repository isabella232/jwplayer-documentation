This article provides an overview of JW Player's Social Sharing plugin as well as instructions for implementation.

Sharing of videos through Facebook, Twitter, blogs and email can dramatically increase reach across the web. See this blog post on sharing for more info and some hints.

About Social Sharing

The JW Player Sharing Overlay displays a customizable layer over the player that allows viewers to do any of the following:

  • Copy the video's embed code
  • Copy a link to the page where the video lives
  • Click to share the video to social networks
  • Complete another action specified by your own custom icon implementation

Users activate the overlay by clicking the Sharing button which, when the Sharing plugin is enabled, appears at the top right of the player when a video is playing.

If the Sharing Overlay is activated while the video is playing, playback is automatically paused. Users can then return to the video by clicking the top right "X", or anywhere else outside the overlay.

When users "share" the video using this overlay, the "link" URL you define (either in the setup block upon embed, or in the content's metadata in the dashboard) is what gets shared.

Dashboard Setup

Go to your JW Player dashboard's Players section and click on the player you would like to edit (or create a new player). In the left-hand accordion menu, you will see a "Sharing" section.

Here, you can turn the Sharing overlay "on" or "off", and set a custom title.

Facebook & Twitter

Code Setup

The sharing functionality is activated by setting up the player using the Sharing configuration block. Inside this block, the following optional parameters can be set:

  • link
    URL to display in the video link field. If no link is set, the URL of the current page is used.
    Example: 
    link: "http://example.com/videos/12345"
  • code
    Embed code to display in the embed code field. If no code is set, the field is not displayed whatsoever. For this, you will need to use an iframe. Example: 
    code: '<iframe src="http://example.com/12345.html"  width="480"  height="270"  frameborder="0"  scrolling="auto" />'
  • heading
    Short, instructive text to display at the top of the sharing screen. The default is Share Video. This is also is displayed as a tooltip for the sharing icon.
    Example: 
    heading: "Custom Sharing Text"
    Note: Since the embed code may contain special characters like <, " or &, you should encodeURI the string in the embed code of your player. The JW Player will automatically decode your string when it is loaded.
  • sites
    New in JW Player 7.2.0, this setting allows for the customization of social icons. By default, we display Facebook, Twitter, and email. However, sites can be added, removed, or reordered based on your preference. Within the sites block, you can specify any number of our built-in sharing options, including:
    • Facebook (facebook)
    • Twitter (twitter)
    • Pinterest (pinterest)
    • Email (email)
    • Tumblr (tumblr)
    • Google Plus (googleplus)
    • Reddit (reddit)
    • LinkedIn (linkedin)
    Example: 
    sites: ['reddit','facebook','twitter']

Custom Sites

If you'd like to create an additional site or action for viewers to access in the Sharing Overlay, specify the following in your sites array:

  • icon
    A URL to the social media icon that you wish to use. Can be in svg or png format.
  • src
    The javascript or URL that you will execute by selecting the particular button.
  • label
    The label that you wish to associate with this sharing method. This will be returned via our API when selected.
Example: 
sites: ['reddit', {icon: 'customicon.png', src: function() {  //some custom javascript  }, label:'mysite'}, 'linkedin']

Media ID Replacement

When using a playlist containing multiple videos, it is possible to set distinct sharing links per item. Therefore, use the string MEDIAID in the code and/or link configuration options. JW Player will, for each video in the playlist, automatically replace the MEDIAID string with the actual mediaid of the video. Here is an example embed:

  var playerInstance = jwplayer("container");
  playerInstance.setup({
    playlist: [{ 
       file: "/videos/12345.mp4",
       mediaid: "12345"
    },{
       file: "/videos/67890.mp4",
       mediaid: "67890"
    }],
    sharing: {
      code: encodeURI("<iframe src='http://example.com/embed/MEDIAID.html' />"),
      link: "http://example.com/page/MEDIAID/"
   }
 });
