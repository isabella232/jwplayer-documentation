This article walks through the ways in which Playlists can be used to play content with JW Player. Mastery of the "Playlists" concept will give you greater control over your JW Player embeds and the content you publish for your users.

For more information about the types of content can be used in a JW Player embed, read about supported formats and basic embedding.

About Playlists

Playlists are lists of one or more pieces of content that can be played, in order, in a JW Player. Playlist objects can either be created in the dashboard, coded by hand in a player setup block, or referenced externally via an RSS feed, XML file, or JSON feed.

When using a Playlist, JW Player 7 automatically includes a visual playlist button on the player's control bar. Mousing over (or touching) this button will display the Playlist items in a scrollable overlay (the "Visual Playlist"). This allows a user to quickly jump between various playlist items. Visual Playlists are rendered entirely on top of the player, without the need for additional space next to (or below) the player.

Visual Playlist Example

The below is an example of what a visual playlist looks like by default in a JW Player (in this example, an RSS playlist is the content referenced in the player setup code). The player populates the Visual Playlist automatically with the title of each item (as specified in the dashboard, or in the embed block itself).

Manual vs. Dynamic Playlists

There are two types of playlists that can be created in the dashboard. They each behave differently, and the differences between them are important.

  • Manual playlists are populated by dragging & dropping videos into the playlist. Manual playlists don't change after they're created unless you change them, and they do not automatically get updated.
  • Dynamic playlists are automatically populated based upon rules. For example, you can create a playlist that follows this rules: Include all videos that are tagged with Cat. If you tag another video with Cat, it would automatically be added to this playlist.

Creating a Playlist

Click the blue New Playlist button to create a new playlist. The dropdown arrow on the right side of this button will allow you to select between the two types of playlists.

Note: make sure you have a couple of videos in your account already, otherwise there will be no content to put inside your playlist.

Managing Playlists

Once a playlist is created, you can view the detail page of that playlist. Here is a screenshot of the detail page for a dynamic playlist (the manual playlist detail page has a similar layout):

On this page, you can manage all aspects of a playlist, including:

  1. The metadata of the playlist, like its title and description.
  2. For Dynamic playlists: The rules for populating the playlist, like the tags to use, the sorting method and the maximum length.
  3. For Manual playlists: Searching and adding new videos by dragging them from the search results.
  4. The current lineup of the playlist. For Dynamic playlist, this is purely informational. For Manual playlists, you can also re-order or delete videos here.

Publishing a Playlist

There are two ways to publish playlists from the dashboard. The simplest one is by publishing it in an existing JW Player from your account. Do this by hitting the Embed button in the left sidebar of the Playlist detail page. This will expand Publish Mode:

Here, you'll see the last player used to publish a video. If you'd like, select a different player you want to use. (Note: this is the same publishing flow outlined in our Basic Embedding article). Preview your playlist in the selected player before embedding by clicking the "preview" button.

Show or Hide the Visual Playlist

While the Visual Playlist is enabled by default, it can also be hidden. When you're setting up your player in the Player Builder page of the dashboard, you'll want to make sure the "Show visual playlist tools" button is un-checked. Click "Save", and you'll see your changes reflected if the player is already embedded.

 

Feeds

The other way to publish or your playlist is through a Feed. Feeds are valuable for syndication of your video content to partners for use or publication elsewhere. Feeds contain all the metadata and media links needed to play your videos.

Both RSS and JSON feed URLs can be found in the "sources" tab of a playlist detail page.

For more about RSS and JSON Feeds, and how to embed them, read our article on external feeds.

It's also possible to use single items of content and configure a playlist with them, directly inside a JW Player embed.

Building a Playlist Directly in the Embed Block

Understanding how a player functions with a single item is integral to understanding how to properly create a multi-item playlst. A single item is used here:

var playerInstance = jwplayer("player");
playerInstance.setup({
  file: 'myvideo.mp4',
  image: 'myposter.png',
  title: 'My Video',
  mediaid: '123456'
});

A Single Item Playlist

Above, we've configured only one single piece of content (an mp4 file) within the player— with a file, poster image, and title. However, let's say that we want to create a single-item playlist. In this case, rather than specifying a file, image, and title directly, we'll be inserting these into a "playlist" config block as a playlist item. Let's look at the above code in the context of a playlist:

playerInstance.setup({
playlist: [{
  file: 'myvideo.mp4',
  image: 'myposter.png',
  title: 'My Video',
  mediaid: '123456'
}]
});

Again, since we're listing this as a single playlist item, both of our code snippets will behave exactly the same way. However, using the playlist block allows us to easily insert additional playlist items if needed.
For reference, the formatting for a playlist is as follows:

playlist: [{}, {}, ...]

A two-item playlist

So, with the above in mind, let's add a second video item to our playlist:

playerInstance.setup({
playlist: [{
  file: 'myvideo.mp4',
  image: 'myposter.png',
  title: 'My Video',
  mediaid: '123456'
},
{
  file: 'mysecondvideo.mp4',
  image: 'mysecondposter.png',
  title: 'My Second Video',
  mediaid: '234567'
}]
});

Awesome! We now have two items in our playlist.

A three-item playlist... and beyond

Any additional playlist items will follow the same convention as above. Each item (meaning a file, image, and title) will exist within {curly} brackets, separated by a comma, and all will be fully contained within a single set of [square brackets], similar to an array. Just for clarity, three playlist items would appear as:

playerInstance.setup({
playlist: [{
  file: 'myvideo.mp4',
  image: 'myposter.png',
  title: 'My Video',
  mediaid: '123456'
},
{
  file: 'mysecondvideo.mp4',
  image: 'mysecondposter.png',
  title: 'My Second Video',
  mediaid: '234567'
},
{
  file: 'mythirdvideo.mp4',
  image: 'mythirdposter.png',
  title: 'My Third Video',
  mediaid: '345678'
}]
});

Any configuration options after a playlist will still require a comma after the second square bracket similar to:

playerInstance.setup({
playlist: [{
  file: 'myvideo.mp4',
  image: 'myposter.png',
  title: 'My Video',
  mediaid: '123456'
},
{
  file: 'mysecondvideo.mp4',
  image: 'mysecondposter.png',
  title: 'My Second Video',
  mediaid: '234567'
},
{
  file: 'mythirdvideo.mp4',
  image: 'mythirdposter.png',
  title: 'My Third Video',
  mediaid: '345678'
}],
primary: 'flash'
});

So again, this is still allowed, just keep the additional comma in mind when embedding!

Sources with a playlist

Lasty, it is entirely possible to create a playlist item that has multiple sources (Either using multiple qualities or fallbacks) in conjunction with our article here. You'll need to nest your sources block in place of "file" within the playlist item. So a quick example of the structure would look like:

playerInstance.setup({
    playlist: [{
        sources: [{source1},{source2},{source3}, ...],
        image: 'myposter.png',
        title: 'My Video',
        mediaid: '123456'
},{
        sources: [{source1},{source2},{source3}, ...],
        image: 'mysecondposter.png',
        title: 'My Second Video',
        mediaid: '234567'
},{
        sources: [{source1},{source2},{source3}, ...],
        image: 'mythirdposter.png',
        title: 'My Third Video',
        mediaid: '345678'
}],
});

An actual example of how this would look with 3 different sources per playlist item:

     mediaid: '123456'
},{
     sources: [{ 
            file: "myFile2.flv"
        },{ 
            file: "myFile2.mp4"
        },{
            file: "myFile2.webm"
        }],
     image: 'image2.jpg',
     title: 'This is another multisource video!',

},{

        },{ 
            file: "myFile3.mp4"
