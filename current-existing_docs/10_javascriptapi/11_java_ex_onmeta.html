When curious about how the player handles certain streams and/or decides which chunk to display, the use of our on('meta') API call can help demystify things.

For this example, we'll be outputting some information about an HLS stream to our console log. We'll need to add the following to our web page in order to see this information:​

playerInstance.on('meta', function(event) {
console.log(event.metadata);
​});

In this output, we're able to see the quality level/resolution of each segment, as well as our perceived bandwidth. This is a great way to confirm that your stream is operating to your liking.

For more advanced features, it may be possible to parse this information. Such information can be utilized to let your viewers know what kind of connection they're getting while watching your stream. Likewise, this information can be fed back to Google Analytics for further tracking. For more information on integrating your Google analytics, you can see the following article located here.

In addition, metadata in either ID3 or EXTINF can be extracted using the same method. This will be included in the metadata object.
