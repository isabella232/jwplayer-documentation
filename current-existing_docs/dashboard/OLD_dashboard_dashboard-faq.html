Players

What's new in JW Player 7?

We've given JW7:

  • Much faster load times
  • New, customizable CSS skins (they're also retina-optimized)
  • MPEG-DASH support
  • ...and lots more (click here to read the full release notes)

 

 

How can I get JW7?

Update your existing saved players to JW7 at any time. If you don't want to update, existing JW6 embeds will continue working for the near future. (Check the Player Detail or Player List pages for version numbers). Click here for more information about updating to JW7.

  • To update existing players to JW7, look for the "update" buttons on the Player Detail and Player List pages.
  • Each new player you create will will be a JW7 player.
  • Or, you can download the JW7 player from Players > Downloads.
Note: If you've done any customizations outside the dashboard with existing players (like custom XML skins, playlists, or any API work), those may be affected by updating — read the full JW7 release notes. In addition, if a large part of your user base relies on Internet Explorer 8, JW7 won't work for them.

 

 

How do saved players work?

Any updates you make to a saved player in the dashboard will be reflected in all the live embeds that use that player.

If you'd like to set overrides in the code at the embed level, you can do that, and those overrides will not change when the auto-updating player is changed & saved. Click here to learn more and to see a code example.

 

What is the Player Library URL, and how do I use it?

The Player Library URL calls the JW Player javascript library PLUS the unique player configurations for that player.

You can place this code in the <head> of your site — this gives your organization this player to use as a default site-wide. It's likely most relevant if you are a developer; click here for more information.


Content

What are the ways I can add content?

You can add content into the dashboard two ways:

  1. Upload videos to host in our system (storage & streaming limits apply)
  2. Add externally-hosted URLs as links in your library (these do not count against hosting limits)

To change which type of content you'd like to add, use the dropdown toggle on the "add new" button in the Content List page.

You may wish to put externally-hosted URLs in your library in order to add tags, metadata, or a poster image.

 

How can I tell which type of content is which?

There are two places where content type is identified. 

  1. On the content list page, in the "type" column
  2. On a content detail page, in the left info bar


 


Embedding

How do I get embed code for a video?

First, configure and save a player in the JW Player Dashboard, or use one of our provided sample players.

Then, get your content ready to go — click here for more about adding content. you may (1) upload content (limits apply), or (2) add externally-hosted URLs as links in your library. To change which type of content you'd like to add, use the dropdown toggle on the "add new" button in the Content List page.

If you're ready to embed:

  1. Click the "embed" button on the piece of content you'd like to publish.
  2. Make sure your desired player is selected in Publish Mode — if it isn't, swap it for the one you want
  3. Copy the provided iframe embed code from the right side of Publish Mode (if you'd like to use a javascript embed, click here for more info), and insert it into your webpage's html.

 

What is Quick Publish?

Get embed code quickly from the homepage by pasting your content URL into the "Quick Publish" box, or by uploading a piece of content from your computer. This will open Publish Mode (which contains the last player you embedded) with your embed code, ready to go!

URLs entered into Quick Publish will also be added to your library as linked content (still hosted externally), in case you'd like to add tags, metadata, or a poster image. Access these from the Content List. Uploads entered into Quick Publish will be added to your library — you can embed them quickly and then go back later to add a custom title, metadata, tags, and more. 

User Permissions

Who can I add as an additional user?

You may add any user who does not already have a JW Player account.

Quick tip:  If you'd like to add a user who already has a separate JW Player account, you can use a "+" symbol with a string in their email address to differentiate it. For example, if "user@example.com" already had a JW Player account, I could add them as "user+1@example.com".
 

What permissions do different users have in my account? 

  • The OWNER has unrestricted access, and is the only person who can delete an account, or transfer ownership of an account to another user. There can only be one Owner per Account.
  • ADMIN users have full account access (including Billing and User Management) — except they cannot change or replace the Account owner, and cannot delete an Account.
  • MANAGERS have access to most site areas — except Billing and User Management

Analytics

How do JW Player analytics work?

Your JW Player Dashboard comes complete with an analytics section, where you can monitor your content performance, and look at activity by device, geography, engagement, and more. Player activity is associated with a unique media ID, to ensure that everything is attributed accurately.

JW Player Analytics are enabled for all accounts by default — as long as your JW Player embeds are set up correctly, we will begin tracking your analytics immediately. If you have any questions about Media IDs, read this. For more information about basic embed setup, click here.

Note: If you already use the dashboard to add content and generate embed code, your setup work is already done.

The following metrics are tracked:

  • Content: Any video or audio content object with a unique Media ID
  • Embeds: Players that have loaded on a webpage
  • Plays: Video starts across all embedded players
  • Completes: Videos watched to completion
  • Daily Unique Viewers: Unique users who have played at least one video
  • Plays per Unique: The number of plays divided by the number of unique viewers on specific day
  • Complete Rate: The number of plays divided by the number of completes, as a percentage
  • Time Watched: Total time watched across all your players
  • Engagement: Use this data to track the performance of your video (where do viewers drop off? which sections do they replay?)

Due to the way that we track analytics from within the player, it is not possible to keep count of viewed time for live streams. Please keep this in mind when looking to track certain aspects of playback.

Ad Tag Previewing

Why doesn't the video preview show the ad tag I configured?

Due to the nature of ad tags, they will not always work in a dashboard environment (various causes might include ad server limitations like: IP limiting, Geo-blocking or Geo-fencing, Impression Caps, and more). We also wouldn't want to impact your ad fill or fill rates with configuration previews.

For these reasons, our video previews will only show JW Player-inserted sample test ads.

In order to truly test your own ad tags, we recommend that you do so in your own test or production environment, or try our Demo Ad Tester Tool.

Self-Hosted vs. Cloud-Hosted Player

What is the Cloud-Hosted Player?

With the Cloud-Hosted Player, we host the JW Player javascript library for you. This ensures a fast, reliable, secure player that is always up-to-date with the latest mobile devices and web standards. The Cloud-Hosted Player's feature functionality is exactly the same as that of the downloadable (Self-Hosted) player, and it has the added bonuses of (1) simple setup in seconds, (2) automatic feature updates, (3) and fast, reliable CDN delivery. All JW Player Dashboard embed codes point to the Cloud-Hosted Player library.

When using the Cloud-Hosted player, you still have control over all available player customizations. For more about customizations, click here.

What is the Self-Hosted Player?

The Self-Hosted Player library is not functionally different from the Cloud-Hosted Player library. It is simply a downloadable version of the player which you can host yourself, on your own servers. The download can be found in the Players > License Keys & Downloads section of your dashboard.

If you are self-hosting our player, you will need to (1) point to your copy of jwplayer.js, and (2) define your self-hosted license key. A code example is available here.

Click here to learn more about the differences between Cloud- and Self-Hosted Players.

Migrating from the Old Dashboard

For more information about JW Player 7, read the full release notes.

What will happen to my existing embeds?

If you're using the cloud-hosted library:
Existing embeds that used JW Player cloud-hosted JavaScript Library will not change at all. We are not auto-updating the existing JW6 library to JW7. If you would like to update your existing pages with JW7, you will need to replace it with a new player library from the new dashboard.
Once you update the player library on your pages, your embeds will be using JW7 and the players might need to be adjusted due to the updates made in JW7.

If you're self-hosting JW Player:
Existing embeds that use a self-hosted JavaScript library will not change at all. To update these to use JW7, download the player zips from your account dashboard, update the license key on your pages, and replace the url to JW Player with the new location where you are hosting JW7.
Once you update the player library on your pages, your embeds will be using JW7 and the players might need to be adjusted due to the updates made in JW7.

What will happen to my analytics data?

The old dashboard's analytics information does not fit into our new data model — this migration paves the way for the updates to come.
