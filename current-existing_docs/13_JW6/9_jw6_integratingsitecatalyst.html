This guide explains how to enable and use the built-in Adobe SiteCatalyst integration with JW Player.

Introduction

JW Player integrates with the Adobe SiteCatalyst (SC) service for providing video playback, engagement and milestone analytics. These video metrics end up next to the regular site metrics, providing an end-to-end picture of user behavior.

The integration is straightforward, leveraging the SC <script> code embedded on the page. This means Adobe SiteCatalyst (including its Media library) must always be available for JW Player to track video events.

JW Player sends a full set of open/play/stop/close commands to SC, providing data for media playback. JW Player does not yet support openAd/click for providing advertising analytics.

SiteCatalyst Setup

To setup successful video measurements, the bulk of the configuration work required is actually on the Adobe SiteCatalyst side. Here is an outline of the steps involved, with links to more info on the blog of Marijka Engel, Adobe's video SME (Subject Matter Expert):

  1. Determining what media variables to record with which events.
    See Creating the Perfect Plan for Video Measurements.
  2. Editing your s_code.js script to add the Media library and media variables.
    See Understanding Video Measurement Implementation.
  3. Configuring the structure and contents of the reports inside the SiteCatalyst console.
    See Initializing Video Measurement Success.
  4. Navigating, understanding and drilling down into these new video reports.
    See Navigating Video Measurement Analytics.

Towards the end of step 2, the JW Player Setup part kicks in. Simply ensure the s_code.js is available on your page and it includes the Media library and media variables. JW Player will pro-actively connect to SiteCatalyst, so no specific JavaScript code is needed on your page.

JW Player Setup

With the Adobe SiteCatalyst library available on the page, it's time for the player. Connect JW Player to SC by adding the sitecatalyst block to your JW Player embed code:

jwplayer("myElement").setup({
    file: "/uploads/example.mp4",
    image: "/uploads/example.jpg",
    sitecatalyst: {} 
});

For a basic setup, this is all that's needed on the player side. JW Player will automatically start pinging open/play/stop/close commands, and Adobe SiteCatalyst will start reporting on playback, engagement and milestones.

SETUP ERROR

If the s_code.js script is not embedded on the page, or the s.Media library is not available, JW Player will render a setup error:

Adobe SiteCatalyst Error:
Could not find Media Module

If you encounter this error, ensure your SC script is correctly configured and embedded on the page. One specific common mistake is that JW Player gets setup before the s_code.js script is embedded. This should always be the other way around.

CONFIGURATION OPTIONS

Two configuration options are available to tune the Adobe SiteCatalyst integration:

mediaName
By default, JW Player always sends the media title property as mediaName to SiteCatalyst. If the title is not set, JW Player uses the file property, stripped of its filepath and querystring. If you want to use another mediaName, use this option to set it.
playerName
By default, JW Player always sends the ID of the player element to siteCatalyst. For the above example setup, the ID would be myElement. If you want to use another playerName, use this configuration option to set it.

Here is an example embed code that sets both configuration options:

jwplayer("myElement").setup({
    file: "/uploads/example.mp4",
    image: "/uploads/example.jpg",
    sitecatalyst: {
        mediaName: "Company Showreel",
        playerName: "Homepage Showreel Player"
    }
});
