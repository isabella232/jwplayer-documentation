Important note: Casting is available as of JW Player 7.0.2. 

JW Player is Google Cast Ready in JW7 for Premium and higher users. This document will cover the functionality currently available and will walk through how to set up a JW Player to be Google Cast-Ready so your viewers can start enjoying content on their big screen! Note that you must be on JW Player 7.0.2 to cast HLS content.

This document applies to users of JW Player Premium. Don't have Premium? Upgrade Now.

Introduction

The Cast functionality will allow your videos to be cast from a Chrome browser via the Chromecast plugin directly to a JW Player receiver app on a Chromecast-connected TV. Once the JW Player detects a Chromecast on the network, a Cast icon appears in the control bar to let viewers know they can display their content on a TV. The JW Player within the browser becomes a remote control for viewing media on a TV screen. Here is a short video demonstrating the process:

Basic Definitions

  • Sender - A Cast enabled JW Player running inside of a Chrome browser. The JW Player requires a sender App ID.
  • Receiver - A JW Player receiver application that runs on the Chromecast device. The receiver application performs the following functions:
    • Provides an interface to display the app's content on the TV.
    • Handles messages from the sender application to control content on the receiver device.
    • Handles custom messages from the sender application that are application-specific (VAST advertising messages and eventually JW Player API support)

Supported Media Formats

The following media formats are supported in the JW Player Chromecast implementation:

  • Video - mp4, WebM
  • Audio - aac, mp3
  • Audio/Video Streaming - M3U8

The following media formats are not supported in JW Player’s Chromecast implementation:

  • RTMP - Will not be supported due to requirement of Adobe Flash. HLS should be used instead.

Registering with Google Cast

In order to use Chromecast on the JW Player you must first register your receiver application with Google. This process takes about 15 minutes to complete. You will be using the JW Player as your receiver application. Please contact support@jwplayer.com to obtain the url for the receiver application.

Steps

  1. Read over the registration process here.
  2. Create a Google Cast SDK Developer Account and be sure to read the TOS and Privacy Policy.
  3. Sign into the Developer Console.
  4. Add a New Application.
  5. Select “Custom Receiver.”
  6. Fill in the form for New Custom Receiver.
    • Name: Your Brand Name.
    • URL: The Receiver URL you need is provided in the Casting section of the Player builder in your dashboard, which you will see when either creating a new player or editing an existing one. If you do not see your Receiver URL, please e-mail us at support@jwplayer.com.
    • Platform: Select Chrome.
    • Icon: Upload a 512x512 icon of your brand logo.
  7. Publish your application.

After your application is published you will receive an Application ID. This takes between five and ten minutes. The ID is used to authenticate with Google that you have signed the terms of service for using Chromecast with your content and turns the Chromecast functionality on in the JW Player.
Additional registration steps can be found on Google's developer site here.

Supported/Unsupported JW Player Features

The following types of embedding options have been verified as functional:

  • RSS playlists
  • Using visual playlist in sender app
  • Playback api calls
  • play, pause, seek
  • HLS Support
The following features are disabled during a cast session:
  • Social sharing overlay
  • Related videos overlay
  • Closed Captions
  • Captions (Planned to be supported in a future release)
  • YouTube video sources cannot be cast
  • HD Quality toggling
  • Currently only the default quality will play on the receiver (Quality toggling is not supported)
  • Quality API
  • Controls API
  • Google IMA advertising plugin
