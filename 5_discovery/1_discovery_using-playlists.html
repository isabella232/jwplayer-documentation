Playlists

While there are certain ways to link to an external RSS or XML file with playlist information, it is absolutely possible to configure a playlist directly into your JW Player embed. Let's jump into a few specific examples.

Creating the player

Understanding how a player functions with a single item is integral to understanding how to properly create a multi-item playlst. A single item is used here:

var playerInstance = jwplayer("player");
playerInstance.setup({
  file: 'myvideo.mp4',
  image: 'myposter.png',
  title: 'My Video',
  mediaid: '123456'
});

A single item playlist

Above, we've configured only one single item within the player with a file, poster image, and title. However, let's say that we want to create a single-item playlist. In this case, rather than specifying a file, image, and title directly, we'll be inserting these into a "playlist" config block as a playlist item. Let's look at the above code in the context of a playlist:

playerInstance.setup({
playlist: [{
  file: 'myvideo.mp4',
  image: 'myposter.png',
  title: 'My Video',
  mediaid: '123456'
}]
});

Again, since we're listing this as a single playlist item, both of our code snippets will behave exactly the same way. However, using the playlist block allows us to easily insert additional playlist items if needed.
For reference, the formatting for a playlist is as follows:

playlist: [{}, {}, ...]

A two-item playlist

So, with the above in mind, let's add a second video item to our playlist:

playerInstance.setup({
playlist: [{
  file: 'myvideo.mp4',
  image: 'myposter.png',
  title: 'My Video',
  mediaid: '123456'
},
{
  file: 'mysecondvideo.mp4',
  image: 'mysecondposter.png',
  title: 'My Second Video',
  mediaid: '234567'
}]
});

Awesome! We now have two items in our playlist.

A three-item playlist... and beyond

Any additional playlist items will follow the same convention as above. Each item (meaning a file, image, and title) will exist within {curly} brackets, separated by a comma, and all will be fully contained within a single set of [square brackets], similar to an array. Just for clarity, three playlist items would appear as:

playerInstance.setup({
playlist: [{
  file: 'myvideo.mp4',
  image: 'myposter.png',
  title: 'My Video',
  mediaid: '123456'
},
{
  file: 'mysecondvideo.mp4',
  image: 'mysecondposter.png',
  title: 'My Second Video',
  mediaid: '234567'
},
{
  file: 'mythirdvideo.mp4',
  image: 'mythirdposter.png',
  title: 'My Third Video',
  mediaid: '345678'
}]
});

Any configuration options after a playlist will still require a comma after the second square bracket similar to:

playerInstance.setup({
playlist: [{
  file: 'myvideo.mp4',
  image: 'myposter.png',
  title: 'My Video',
  mediaid: '123456'
},
{
  file: 'mysecondvideo.mp4',
  image: 'mysecondposter.png',
  title: 'My Second Video',
  mediaid: '234567'
},
{
  file: 'mythirdvideo.mp4',
  image: 'mythirdposter.png',
  title: 'My Third Video',
  mediaid: '345678'
}],
primary: 'flash'
});

So again, this is still allowed, just keep the additional comma in mind when embedding!

Sources with a playlist

Lasty, it is entirely possible to create a playlist item that has multiple sources (Either using multiple qualities or fallbacks) in conjunction with our article here. You'll need to nest your sources block in place of "file" within the playlist item. So a quick example of the structure would look like:

playerInstance.setup({
    playlist: [{
        sources: [{source1},{source2},{source3}, ...],
        image: 'myposter.png',
        title: 'My Video',
        mediaid: '123456'
},{
        sources: [{source1},{source2},{source3}, ...],
        image: 'mysecondposter.png',
        title: 'My Second Video',
        mediaid: '234567'
},{
        sources: [{source1},{source2},{source3}, ...],
        image: 'mythirdposter.png',
        title: 'My Third Video',
        mediaid: '345678'
}],
});

An actual example of this would look like with 3 different sources per playlist item is shown here:

playerInstance.setup({
playlist: [{
     sources: [{ 
            file: "myFile.flv"
        },{ 
            file: "myFile.mp4"
        },{
            file: "myFile.webm"
        }],
     image: 'image.jpg',
     title: 'This is a multisource video!',
     mediaid: '123456'
},{
     sources: [{ 
            file: "myFile2.flv"
        },{ 
            file: "myFile2.mp4"
        },{
            file: "myFile2.webm"
        }],
     image: 'image2.jpg',
     title: 'This is another multisource video!',
     mediaid: '234567'
},{
     sources: [{ 
            file: "myFile3.flv"
        },{ 
            file: "myFile3.mp4"
        },{
            file: "myFile3.webm"
        }],
     image: 'image3.jpg',
     title: 'This the final multisource video!',
     mediaid: '345678'
}]
});

 

 

This article introduces the playlist management capabilities of JW Platform. These can be found in the Playlists section of the dashboard:

This document applies to users of JW Platform. Don't have Hosting? Upgrade Now.

Creating Playlist

Hit the big Create New Playlist button to popup the playlist overlay. In this, set a title for your playlist and the type op playlist you want to create:

The two types of playlists behave differently, so make sure you understand their details:

  • Manual playlists are populated by hand. You literally drag & drop videos into the playlist, just like you would with a tool like iTunes or Winamp. Manual playlists won't automatically get updated.
  • Dynamic playlists are automatically populated based upon rules. For example, you can create a playlist that has All videos that are tagged with Cat. Every time you tag a new video with Cat, it will automatically get added to that playlist.

Note: make sure you have a couple of videos in your account already, otherwise there's no content to put inside your playlist.

Managing Playlists

Once a playlist is created, you can jump to the properties page of that playlist. Here is a screenshot of the properties of a dynamic playlist (manual playlist properties are very similar):

On this page, you can manage all aspects of this playlist, including:

  1. Updating the metadata of the playlist, like its title and description.
  2. For Dynamic playlists: Updating the rules for populating the playlist, like the tags to use, the sorting method and the maximum length.
  3. For Manual playlists: Searching and adding new videos by dragging them from the search results.
  4. Viewing the current lineup of the playlist. For Dynamic playlist, this is purely informational. For Manual playlists, you can also re-order or delete videos here.

The right sidebar of the page contains the options for publishing the playlist to your audience.

Publishing Playlists

There are two ways to publish playlists from JW Platform. The most important and widely used one is by publishing it in JW Player. Do this by hitting the Embed This Playlist button. It will popup the publishing overlay:

Here, select which player design you want to use to publish your playlist. Next, copy/paste the resulting embed code into your own website or blog. A preview of your video+playlist is shown below the code.

Use the small links at the bottom to quickly publish the playlist to Facebook, to Twitter, or to Email in a preview page.

RSS Feed

The other way to publish playlists is through RSS Feeds. This method is valuable for syndication of your video content to partners that have their own video player. The RSS feed contains all the metadata and media links they need to play your videos.

RSS Feeds are also useful for syndicating your content to podcasting services like iTunes. Our feeds are standard RSS and compliant with many of them.

JSON Feed

The latest addition to JW Platform allows for a user to receive a JSON feed of a particular video or playlist object. Compared to existing RSS feeds, JSON feeds allow for information related to:

  • Available Transcodes
  • Transcode labels
  • Media dimensions
  • Video type
  • Content Duration
  • Passthrough files
  • Available Caption/Thumbnail Tracks

With the above information, JW Platform's JSON feeds can be used to verify the completion/quantity of transcodes, as well as to populate meta information for social media sites like Facebook or Twitter. Previously, this information was only accessible via JW Platform's API, but it can now be pinged on a production site, which does not apply to JW Platform's 10 API calls/minute limit.

 

 

This article explains how to easily display the contents of your playlist with JW Player 7's integrated visual playlist feature.

Introduction

When using a playlist in any format(setup, xml, RSS) JW Player 7 will automatically include a visual playlist button on its control bar. Mousing over (or touching) this button will display a visual playlist, allowing a user to quickly jump between particular playlist items. Visual playlists are now rendered entirely over the player, without the need for additional vertical or horizontal space.

The visual playlist can be turned off by setting visualplaylist to false within the configuration setup. This is true by default.

Example

The below is an example of the player's default handling of a visual playlist. The only thing specified in this case is an RSS playlist. The player will then populate the visual playlist automatically with the specified titles of each item.

 

Embed code

<div id="myElement"></div>

<script>
var playerInstance = jwplayer("myElement");
    playerInstance.setup({
      playlist: "/uploads/myPlaylist.rss"
    });
</script>

Customizing the Visual Playlist with CSS

The default style of the visual playlist has a maximum height. This is so it does not take up the entire height of the player. The visual playlist can be easily customized with CSS.

Here is some CSS you can add on your page or adjust directly in your self-hosted seven.css to make the visual playlist wider and as tall as the player.

/*Sets the visual listbar width to 400 px and realigns it above the icon. */
 .jw-skin-seven .jw-overlay .jw-playlist-container {
  width:400px;
  left:-20%;
 }

/* overrides the player settings for how tall the visualplaylist can be. 195px is the about right for a player that is 270px tall, including the controlbar height, and menus. */
.jwplayer .jw-playlist {
min-height: none;
max-height: none;
height: 195px;

 }

/* Makes it so the list items are left aligned in the visualplaylist */
 .jwplayer .jw-playlist .jw-option {
  padding: 0 15px;
  margin: auto;
  text-align: left;
}
