The JW Player you know and love has taken a giant leap forward! We’ve carefully re-designed and re-engineered the entire player — from the core API through the visual interface — to build the fastest, most powerful way to deliver your videos to your viewers. The following document should be quite useful if you are planning on migrating your JW6 players to use JW7, as it goes over settings that have been changed or removed.

For a list of features that have been added, please see our JW7 release notes.

General Changes

  • JW 7's interface is rendered in HTML regardless of media provider. Flash media playback will get HTML controls.
  • Analytics.js is the only supported method of Google Analytics usage.
  • The player will produce a setup error if it takes longer than 10 seconds to load.
  • Minimum version of Adobe Flash Player required for JW Player is now 11.2
  • YouTube will only render via iFrame API in HTML5.
  • Semantic naming of player versions and plugins.

Removed Features

Compared to JW6, the following functionality has been removed entirely from JW Player 7:

  • XML-based skinning model - This has been replaced with CSS skinning
  • Integrated Adobe Sitecatalyst support - Removed in favor of Custom API tracking
  • The JW6 listbar - Replaced by JW7's Visual Playlist
  • Support for Internet Explorer 8 - Microsoft has announced that it is ending IE8 support on January 12, 2016
  • Flash cookies - No longer used for storing user settings
  • Pseudostreaming for Flash-based media playback
  • Download fallback and the fallback configuration option
  • The JW Player Wordpress Plugin does not support JW Player 7 at this time

License Key Changes

With the introduction of JW7's new Free features, there have been some changes made to our existing license key structure.

  • JW7 requires a license key, regardless of using a free or paid version. You will receive a setup error if you fail to configure a jwplayer.key on your page.
  • JW Player 7 will not work with a JW6 license key. You can find your new JW7 license key on your account page.
  • With JW7, Pro features have been moved to Free. As a "Thank You!" to existing Pro users, the use of JW7 Pro keys will unlock Premium features in JW7.

For any additional licensing concerns, please see our License and Payments FAQ.

API Changes

The API is core to JW Player extensibility, and JW Player 7 makes it even easier and friendlier to use. By converting to an interface based on the model–view–controller (MVC) application design paradigm. All up-to-date API documentation can be found in our Javascript API Reference.

  • All onEvent() methods have been deprecated in JW7 and should be replaced with on(‘event’).
  • .container was removed. getContainer() should be used instead.
  • dispatchEvent should no longer be used, it is replaced with trigger(‘event’)
  • removeEventListener should no longer be used, it is replaced with off(‘event’)
  • All player states will be returned in lower case.

For more information regarding core changes to our API, visit our API Quickstart Guide.

Skinning Changes

Now you can craft robust, branded video player designs with CSS, the standard web styling language! It’s straightforward, looks stunning on retina screens, and even makes the player load faster.

  • XML skins are no longer supported in JW7 and should be remade in CSS.
  • If an XML file is specified as a skin, we will load our seven skin will load instead.

All JW6 Premium skins have been remade into CSS and are packaged as part of the core player. If you had an embed with a JW6 Premium skin name, the player will automatically switch that that skins css. See CSS Skinning and Styling for a full guide on how to get up and running.

Flash as a Video Provider

To provide the most flexibility with skinning and playback, Flash is only used as a means to play media. This means that when the player renders in Flash, it is no longer an entirely separate video player. This allows JW7 to intelligently switch between Flash and HTML5 rendering modes within a single playlist. All plugin work needs to only be done in HTML. Setting primary to either Flash or HTML5 will still cause playlist items to render consistently when possible. Note that even when displaying media via Flash, the control bar and icon overlays will continue to render in HTML5.
