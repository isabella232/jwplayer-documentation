When the JW Player Support Team is helping to troubleshoot a JW Player issue (direct support available for JW Player Premium and above customers only), it helps us a great deal to have a reproduction page (also known as a test page). This test page allows us to do a few things:

  • Reproduce the issue you are seeing (this helps us rule out browser, device, or firewall issues).
  • Cleanly determine the cause of your issue, with no extra code in the way
  • Send you back a link to an updated page, if we have a code suggestion or correction
  • Escalate the issue internally, if necessary
We can usually see what you are trying to do by reading your code, but it is also helpful to know:
  • Your exact reproduction steps
  • What you are expecting the player to do
  • What the player is actually doing

By using a basic embed of our player, we can help isolate the part of your code that is causing the problem. Completing this step properly will save time for you and for our support team, getting your videos back up and running as soon as possible.

 

What does a good test page include?

  • The JW Player Javascript library URL you are using to load our player, and if necessary, the self-hosted license key
  • The HTML <div> tag for our player
  • Your jwplayer().setup()
  • List of reproduction steps, expected behavior, and actual outcome

 

While we are experts when it comes to our player, we cannot help you debug any code on the page that does not belong to us. Therefore, it is also important to note that there are a few things that should not be included in your reproduction page.

 

What does a good test page NOT include?

  • Any other Javascript code or libraries not required for our player, like jQuery or require.js
  • Extra CSS code or stylesheets 

 

Test Page Template

Here is a template for you to use when making a reproduction page:

<!doctype html>
<html>
<head>
<script src="http://your_cloud-hosted_player_URL_or_self-hosted_jwplayer.js"></script>
<!-- if necessary: <script>jwplayer.key="your_self-hosted_license_key";</script> -->
</head>
<body>
    <b>Issue:</b> Please indicate issue here
    <br><br>
    <b>Reproduction steps:</b> Please state the steps necessary to reproduce this issue
    <br><br>
    <b>Expected behavior:</b> Please state your expected behavior here
    <br><br>
    <b>Actual outcome:</b> Please state the actual outcome here
    <br><br><br>
    <div id=player>Loading your player...</div>
    <script>
        jwplayer('player').setup({
            file: 'your_video_URL',
        });
        
        //include any of your Javascript API calls here 
    </script>
</body>
</html>
