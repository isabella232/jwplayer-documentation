Important note: Casting is available as of JW Player 7.0.2. 

Embedding

As of JW Player 6.12, HLS content can be cast in Flash mode. MP4 content can be cast in either Flash or HTML5 mode. Please note, the cast configuration block must be included in the embed script in order to detect and begin communication with a Chromecast device. As with all casting implementations, the Chrome browser must be used to cast content.

This document applies to users of JW Player Premium. Don't have Premium? Upgrade Now.

Setting up the Cast configuration

The cast configuration block turns on the mechanism the JW Player uses to interface with a Chromecast device. It also contains the customization options to configure how the Player appears on the TV screen.

The following is a sample configuration block used to enable Chromecast:

cast:{
    appid:”[GOOGLE APP ID]”,
    loadscreen:”/img.png”,
    endscreen:”/img.png”,
    logo:”logo.png”,
    railcolor:”#e31e76’
}

Configurable Chromecast options

appid
This is the only required field to enable Chromecast. This is the ID obtained from registering your application with Google through the process outlined in the Introduction to Casting support section. Note: If you do not enter a valid App id, a Cast session will not be started.

loadscreen
This is an optional image that will display on the TV once the connection between the sending app and receiver app is made. This should be an image file. The JW Player will expand the image to fit the resolution of the screen that is being cast on. If no image is specified, the JW Player provides a default screen. The load screen is visible for a minimum of 3 seconds and then until the media is ready to be played.
The recommended image size is 1280px x 720px. It is recommended that file sizes be kept smaller to provide quicker load times (jpg, png, gif, etc.).

endscreen
This is an optional image that will display on the TV once all videos have completely played. This should be an image file. The JW Player will expand the image to fit the resolution of the screen that is being cast on. If no image is specified, the JW Player provides a default end screen image. The end screen is visible on the TV screen for a minimum of 150 seconds.

railcolor
The railcolor field allows you to specify a hex color value to customize the display of the timeslider on the receiver JW Player App.

logo
The logo allows you to place your own branded logo in the top right corner of the receiver app.

A note on HLS and Chromecast CORS

As mentioned on our helpful crossdomain page here, accessing resources from a particular domain requires a properly configured crossdomain policy from that domain. In the case of Chromecast HLS, the receiver app URL is the page that will request a manifest.

Because of this, you must allow our receiver app domain access to the site hosting your M3U8. The domain to allow is:

https://jwpsrv.com

Note that this will require CORS configuration, and not crossdomain.xml, since Chromecast loads manifests via HTML5. For more information on configuring CORS, please see the helpful site here.
